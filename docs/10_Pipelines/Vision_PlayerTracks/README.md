# 🧍 球员检测与跟踪管线（Vision → Player Tracks）

## 目标 / 范围
从视频生成稳定的球员轨迹（tracklets），为后续合并/过滤与融合定位器打底。

## 输入
- 视频（当前跟球多；未来 A/B 固定机位为主）
- 检测模型（现有 best.pt 需持续提升；未来通用预训练+自采微调）

## 输出（关键工件）
- 逐帧检测结果（bbox/score）
- 初级轨迹 tracklets（带ID、时间范围、轨迹点）
- 回归集（10~20 clips）+ 一键评估脚本（强烈建议）

## 依赖（上游/下游）
- 上游：采集质量、标定可选
- 下游：轨迹合并过滤、球员世界轨迹融合、球似然场特征

## 当前进度 & 已知问题
- ✅ 跟踪→合并→过滤管线已能跑骨架
- ⚠️ 模型精度与视角不匹配导致漂移/断裂；缺少固定回归评测协议

## 指标 / 验收标准
- track 数量、平均时长、ID switch、漂移率（先简化也行）
- 回归集对比输出可复现（每次改动都能对比）

## 下一步（可执行）
- 立刻建立回归集（10~20 clips），固化“每次必跑”
- 网上找同类固定机位素材预训练，等自采后微调
