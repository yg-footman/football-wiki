# 📡 定位器 → 球员世界轨迹管线（Tag → Player World Trajectory）

## 目标 / 范围
用球员定位器提供高可信世界轨迹基准，与视觉轨迹融合/校准。注意：球无定位器。

## 输入
- 球员定位器时序：时间戳、位置/速度/航向等
- 场地世界坐标系定义（world_geometry）

## 输出（关键工件）
- 每球员整场世界轨迹（可追溯、可复现）
- 硬件测试 SOP（室外基准、记录项、判定标准）

## 依赖（上游/下游）
- 上游：硬件稳定性、时间戳可靠
- 下游：球轨迹推断约束、战术指标、论文评测

## 当前进度 & 已知问题
- ⚠️ 曾遇到电平/边沿异常、乱码、接触不良疑似；室内搜星 0 属正常
- ✅ 方向明确：先定义“最小可用闭环”，用室外空旷环境做基准

## 指标 / 验收标准
- 稳定上报、时间戳单调、丢包率低于阈值
- 基准测试可复现（同场景结果稳定）

## 下一步（可执行）

- 每次测试写实验记录（避免重复踩坑）
